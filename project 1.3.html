3. IDEATE

After defining the core problem and understanding Obinna’s needs, I moved into the ideate phase. This phase focused on exploring multiple ideas and narrowing them down to practical, user-centered solutions.

USER-FLOW

(Onboarding - Registration [First time user])

→

Step 1

After installing and opening the app, the user arrives at the welcome screen which after a 3 seconds delay or with a tap on the screen goes to the next screen.

→

Step 2

The user is taken to the intro screen which briefly. The intro screen is 3 in number.

he user can either click ‘Next’ to see the next intro screen or click ‘Skip’ the intro screen.

Step 3

The user is prompted to enter their email and password. They can confirm does details and proceed.

The user can also register using their google, face-book or twitter account.

→

Step 4

After registration, the user is taken to the ‘Role Selection’ screen in which they are to select whether they are a client/customer user or a serviceman.

→

Step 5

The user is then prompted to create a client user profile.

This can not be skipped.

Step 6

The user would then need to verify their phone number.

→

Step 7

The user would then need to add their location either automatically or manually.

→

Step 8

The user can add their photo or skip that and move forward.

Step 9

Congratulations. The user has successfully created their profile and can access the home screen and book the service the need.

USER-FLOW

(Booking of service)

→

Step 1

From the ‘Our Services’ section, user can click on service of choice. User is then taken to the ‘Book Appointment’ screen where they select the date and time the need the services.

→

Step 2

After selecting date and time, user clicks on ‘Continue’ which takes them to the next screen. The screen shows a list of servicemen ‘Available’ and ‘Busy’ that day.

The user can also use the filter icon to show only ‘Available’ servicemen.

Step 3

The user then clicks on serviceman of choice which takes them to the serviceman’s profile.

On the profile, user can check the reviews of the serviceman or read more about the serviceman. If satisfied, user can go ahead and click on ‘Book Now’ to continue the booking process.

→

Step 4

On the screen, user is given the option to a note describing the issues in details. User can also include photos if necessary. This however is optional but recommended.

→

Step 5

After describing the issues or choosing not to, user can click on ‘Continue’ and proceed to select payment method.

Step 6

After selecting payment method (card payment), user clicks on ‘Continue’ to the screen where user fills card details and clicks on ‘Add card’. User is taken to the next screen and they click ‘Continue’ which takes them to the ‘Review Summary’ screen.

Step 7

Where user reviews the information and is satisfied, they can click on ‘Continue’ which takes them to the next screen with an information that reads ‘Booking request sent’. An additional information is also provided which lets the user know they will receive a notification if serviceman accepts, rejects or reschedules offer.
